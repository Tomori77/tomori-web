# Tomori Web 改动溯源

## 版本

- 目标版本：`v0.2.0`
- 记录时间：2026-07-13

## 用户原话

> 1.超级管理员仪表盘的审核日志改为人类可读
> 2.管理后台的用户管理的4个用户组现在内容条目显示过大，将里面的用户显示上下宽度缩窄一半，左右宽度不出现滑条
> 3.在我的文章页面左侧放置一个收信箱，专门用来收集与当前用户有关的文章提醒，与公告一样红点提醒，打开我的文章就消除提醒。
> 3.因为板块删除造成的文章转移发送的通知不再放入个人资料的公告栏，放入我的文章中的收信箱，若有多篇文章被移动则合并为一条提醒。
> 4.文章审核成功或者被驳回在收信箱中给出提示。
> 5.我的文章中的收信箱最多保留10条提醒。
> 6.我的文章中的每篇文章条目上下宽度变为现在的一半，摘要仅显示一行，超出部分用...省略。

## 系统改动

- 超级管理员仪表盘的审核活动流增加中文动作名称和人类可读描述，审核通过、驳回及驳回原因直接展示。
- 用户管理四个分组固定为管理员、作者、普通用户、封禁名单，四栏无横向滚动，每栏使用有限高度的纵向滚动区域，用户卡片和文字高度压缩。
- 新增 `article_notifications` 独立文章提醒表和 `migrations/006_add_article_notifications.sql`。
- “我的文章”页面新增左侧文章收信箱，最多加载10条提醒。
- 文章提醒未读数加入用户会话数据，导航栏“我的文章”显示未读红点。
- 打开“我的文章”页面后批量标记当前用户的文章提醒为已读并清除红点。
- 文章审核通过或驳回时向作者收信箱写入提醒，驳回提醒包含审核原因。
- 板块强制删除导致的文章迁移通知改写入文章收信箱，不再写入个人资料公告栏；同一作者的多篇迁移合并为一条提醒。
- 旧的板块迁移公告通过迁移转换到文章收信箱，避免历史通知继续出现在个人资料公告栏。
- 我的文章条目高度压缩，摘要限制为单行并使用省略号截断。
- 个人资料公告与文章收信箱保持数据和未读状态分离。
